Declaration of Principles


We are using the following principles to guarantee the greatest long-term experience for people/companies using Yanel.

  • Eat our own dogfood
  • Continuous Integration and Testing
  • Performance (related: Caching)
  • Scalability
  • Redundancy/Clusterable
  • Forward and Backwards Compatibility
  • Simplicity
  • Out-of-box usage (minimal setup)
  • Package/Dependency manager
  • Upgradeability
  • Security
  • I18n/Localization
  • Integrated Documentation
  • Logging/Debugging during production
  • Rapid prototyping/development
  • Component based/Modularity
  • Release Planning/Roadmap/Predictability
  • Offline
  • Watching and Listening to the user
